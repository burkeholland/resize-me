ResizeMe — Ultra-Minimal UI

The guiding principle: this window is opened rarely. Users interact with ResizeMe through the tray icon and hotkey 99% of the time. When they do open settings, they should see everything at a glance — no scrolling, no navigation, no separate pages.

Design Principles

  • Everything visible at once — no tabs, no navigation, no scrolling needed at default size
  • Immediate-apply — every change takes effect instantly. No Save/Revert/action bar.
  • Presets are the star — the preset list dominates. Behavior settings are secondary.
  • Read-only display, edit-on-demand — presets show as clean read-only rows; editing happens via flyout/dialog
  • Small window — 480 × 520px. Feels like a utility popover, not a full app.

Anatomy — What Changed & Why

Hero area replaces page header + "active preset" card

Instead of a big "ResizeMe" title + description + a separate "Active preset" card (3 visual elements), we combine everything into a single compact hero: the active preset name as a Subtitle (20px Semibold), dimensions + hotkey badge below it, and the "Resize now" button on the right. One row replaces three.

Space saved: ~80px of vertical space eliminated. The user instantly sees what matters: what preset is active, what hotkey triggers it, and a button to test it.

No navigation rail

Removed entirely. The 200–320px nav pane consumed a third of the window for 3 items. Everything fits in a single column now.

Preset rows — radically simplified

Each preset is now: Radio + Name + Dimensions + Delete button. That's it.

  • No inline editing — names and dimensions are display-only. Click a preset row to edit (opens a small flyout).
  • No reorder buttons — the ↑/↓ buttons added 64px of horizontal clutter per row. If ordering matters, drag-and-drop is better UX (but we can skip it for v1).
  • No labels above inputs — "Name", "Width", "Height" column headers are gone. The layout is self-explanatory.
  • Active row gets subtle blue tint — rgba(0,95,184,0.04) — the same pattern WinUI uses for selected items.

No descriptions on behavior cards

The old UI had description text under every toggle: "Center on the current monitor", "Start ResizeMe when you sign in." These are unnecessary — the labels are self-explanatory. Removing descriptions saves ~16px per card and reduces visual noise.

No floating action bar

Removed. All settings apply immediately:

  • Selecting a radio button → saves active preset immediately
  • Toggling a switch → saves immediately (already works this way)
  • Editing hotkey → saves on blur or Enter
  • "+ Add preset" → opens a ContentDialog with Name, Width, Height fields

"+ Add preset" as a text link

A WinUI HyperlinkButton-style link sits below the preset list. Much lighter than a full button. Opens a dialog.

Add Preset Dialog

When the user clicks "+ Add preset", a WinUI ContentDialog appears. This is the only place where text inputs for name/width/height live.
